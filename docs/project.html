Stages of development

(This is also briefly mentioned in the features page.)

Stage 1 - Basic terminal functionality

Not the most exciting, but still a start. This would be when you can change directories, create them, list directory contents, echo text, run updates, create files, list a file's contents, delete files and directories, and the like.

Stage 2 - Basic text editor

This is tricky. Either I'll try to package in an editor, build my own, or add something super minimalistic, like a simple input/output system.

Although, when I first thought about this problem, a thought popped into my head:
What about the user's choice?

Stage 2 and 1/2 - Colors!

Once again, something tricky. As you probably inferred, this would add some text coloring, and maybe bring ECTerm out of the native terminal runtime. I could probably use a GUI crate, but maybe there's some funky text-coloring library out there (side note: I hear you, Linux users, and I know that I could probably use the built-in colors from the terminal, but I want ECTerm to bring some color to Windows, too :)).


An example of a colored terminal on Arch Linux.
Average Linux User

Stage 3 - Package manager

You remember when I said the text editor would be tricky? Well, this would be a nightmare.
...kind of.

There are a few ways I could approach this (ranked from easiest to hardest):

  1. Having a repository with packages in it
  2. Adding funtionality for getting binaries from crates.io
  3. Adding funtionality for getting packages from other repositories
  4. Creating a site presicely for hosing ECTerm packages (though maybe for other uses as well)

The most convenient and flexible way to get packages would probably be from third-party (or any) repositories from something like GitHub.

History

So, I bet you've been wondering: ECTerm? What is that supposed to mean? Glad you asked! It's more elaborate than this, but I hope this will do well to explain:

ECTerm was originally from a set of Python programs, that was called ECCellsior (although before that, it was called EC.OS, though since it barely resembled an OS, I renamed it). ECCellsior and ECTerm are both acronyms, coming from my own fictional orginization, ECTAC, which stood for Earth Clone Troopers And Company.

This is more elegantly explained in ECTerm's old README.md (now accordingly named OLD.md).

So there you have it!

I probably could come up with some more things to add, but for now, what I have is enough. I will commit my best to ECTerm, and I hope that people will be able to enjoy it!
