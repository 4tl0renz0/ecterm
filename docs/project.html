The Project

This page is probably going to be the most elaborate yet.

This page will contain information about the ECTerm repository, the planned out stages of development, and how ECTerm came to be.

The Repository

The ECTerm git repository can be found here, on GitHub, which is a git-based code hosting website, and hence, where the magic is held.

(Side note: You can find this website in the docs folder and open a pull request if there's an issue.)

Again, the repository is where you'll find the latest releases and updates, as well as all the source code.

Stages of development

(This is also briefly mentioned in the features page.)

Stage 1 - Basic terminal functionality

Not the most exciting, but still a start. This would be when you can change directories, create them, list directory contents, echo text, run updates, create files, list a file's contents, delete files and directories, and the like.

Stage 2 - Basic text editor

This is tricky. Either I'll try to package in an editor, build my own, or add something super minimalistic, like a simple input/output system.

Though when I first thought about this problem, a thought popped into my head:
What about the user's choice?

Stage 3 - Package manager

You remember when I said the text editor would be tricky? Well, this would be a nightmare.
...kind of.

There are a few ways I could approach this (ranked from easiest to hardest):

  1. Having a repository with packages in it
  2. Adding funtionality for getting binaries from crates.io
  3. Adding funtionality for getting packages from other repositories
  4. Creating a site presicely for hosing ECTerm packages (though maybe for other uses as well)

The most convenient and flexible way to get packages would probably be from third-party (or any) repositories from something like GitHub.

So there you have it!

I probably could come up with some more things to add, but for now, the list I have is enough. I will commit my best to ECTerm, and I hope that people will be able to enjoy it!
